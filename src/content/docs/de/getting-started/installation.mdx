## Systemanforderungen

### Mindestanforderungen

| Komponente | Anforderung |
|-----------|-------------|
| OS | Linux, macOS, Windows |
| Rust | 1.95+ |
| Memory | 4 GB RAM |
| Disk | 2 GB freier Speicherplatz |

### Empfohlen

| Komponente | Anforderung |
|-----------|-------------|
| Rust | 1.95+ (latest supported) |
| Memory | 8 GB RAM |
| CPU | Mehrkernprozessor |

## Rust installieren

Falls Rust noch nicht installiert ist:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

Setzen Sie die richtige Toolchain-Version:

```bash
rustup default 1.95
```

Fügen Sie das WASM-Ziel zum Bauen von Komponenten hinzu:

```bash
rustup target add wasm32-wasip2
```

## GTC CLI installieren

Installieren Sie zuerst den `gtc`-Router und lassen Sie danach `gtc install` die Greentic-Begleitwerkzeuge installieren. Installieren Sie `greentic-cli` oder einzelne Begleit-Crates nicht direkt für den normalen Gebrauch; `gtc install` hält die delegierten Binärdateien passend zum Router.

<Tabs>
<TabItem label="cargo-binstall (Empfohlen)">

```bash
cargo install cargo-binstall
cargo binstall gtc
gtc install
gtc doctor
```

</TabItem>
<TabItem label="Aus dem Quellcode">

```bash
git clone https://github.com/greenticai/greentic.git
cd greentic/greentic
cargo build --release

# Add to PATH
export PATH="$PATH:$(pwd)/target/release"
gtc install
gtc doctor
```

</TabItem>
<TabItem label="Vorgebaute Binärdateien">

Von [GitHub Releases](https://github.com/greenticai/greentic/releases) herunterladen:

```bash
# macOS (Apple Silicon)
curl -LO https://github.com/greenticai/greentic/releases/latest/download/gtc-darwin-arm64.tar.gz
tar -xzf gtc-darwin-arm64.tar.gz
sudo mv gtc /usr/local/bin/

# macOS (Intel)
curl -LO https://github.com/greenticai/greentic/releases/latest/download/gtc-darwin-x64.tar.gz

# Linux (x86_64)
curl -LO https://github.com/greenticai/greentic/releases/latest/download/gtc-linux-x64.tar.gz

# Windows
# Download gtc-windows-x64.zip from releases
```

</TabItem>
</Tabs>

## Entwicklungswerkzeuge

### Greentic Toolchain

Die delegierten Greentic-Binärdateien installieren oder aktualisieren:

```bash
gtc install
```

Mit `gtc install --dry-run` können Sie den Installationsplan prüfen. Für kontrollierte Umgebungen kann die Toolchain gepinnt werden:

```bash
gtc install --release 1.0.15
gtc install --manifest ./greentic-toolchain.json
```

### Komponenten-Erstellung

`gtc install` installiert die Standard-Begleitwerkzeuge. Für eigenständige Komponentenentwicklung kann zusätzlich `cargo-component` installiert werden:

```bash
cargo install --locked cargo-component
```

Danach nutzen Sie die delegierte Komponentenoberfläche:

```bash
gtc dev component --help
```

## Ihr Setup verifizieren

Führen Sie Folgendes aus, um zu prüfen, dass alles funktioniert:

```bash
# Check GTC CLI
gtc --help

# Check Rust WASM target
rustup target list --installed | grep wasm32-wasip2

# Create and run a test bundle
gtc wizard --dry-run
```

## Fehlerbehebung

### Häufige Probleme

**Error: "wasm32-wasip2 target not found"**
```bash
rustup target add wasm32-wasip2
```

**Error: "Cannot connect to NATS"**
```bash
# Start embedded NATS with gtc start
gtc start ./my-bundle --nats on
```

**Error: "Permission denied"**
```bash
# On Linux/macOS, ensure binary is executable
chmod +x /usr/local/bin/gtc
```
