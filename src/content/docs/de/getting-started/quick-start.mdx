Diese Anleitung bringt Sie in wenigen Minuten von null zu einem laufenden Greentic digital worker. Der schnellste Weg ist, eine der veröffentlichten Demos aus dem Katalog [`greentic-demo`](https://github.com/greenticai/greentic-demo) zu starten. Jede Demo verwendet denselben dreistufigen Ablauf.

<Aside type="tip">
Möchten Sie alle verfügbaren Demos durchsuchen und genau sehen, was jede einzelne tut? Wechseln Sie zu [Demos ausführen](/de/getting-started/running-demos/).
</Aside>

## Voraussetzungen

Bevor Sie beginnen, stellen Sie sicher, dass Sie Folgendes haben:

- **`gtc` CLI** installiert — siehe [Installation](/de/getting-started/installation/)
- **Rust** 1.91 oder neuer (nur erforderlich, wenn Sie aus dem Quellcode bauen)
- Eine **öffentliche URL** für Webhook-Provider (z. B. `ngrok` oder `cloudflared`) — für den Quickstart nur mit WebChat nicht erforderlich

## Die quickstart-Demo starten

Die Demo `quickstart` ist das kleinste End-to-End-Bundle: eine Begrüßungskarte, eine Info-Karte und eine Echo-Interaktion über WebChat. Sie benötigt keine externen Konten.

<Steps>

1. **Erstellen Sie das Bundle** aus dem veröffentlichten Create-Answers-Dokument:

   ```bash
   gtc wizard --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-create-answers.json
   ```

   Der Wizard lädt das Demo-Pack herunter, löst Capability-Abhängigkeiten auf (z. B. zieht er `state-memory` automatisch hinzu, weil der WebChat-Provider `greentic:state/state-store` benötigt), und schreibt ein lokales Bundle-Verzeichnis.

2. **Konfigurieren Sie Provider** mit dem veröffentlichten Setup-Answers-Dokument:

   ```bash
   gtc setup ./quickstart-demo-bundle --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-setup-answers.json
   ```

   Das Setup erkennt den Tenant des Bundles automatisch, speichert Secrets pro Provider und validiert, dass jede erforderliche Capability erfüllt ist.

3. **Starten Sie die Runtime**:

   ```bash
   gtc start ./quickstart-demo-bundle
   ```

   Der Runner gibt die öffentliche WebChat-URL aus. Öffnen Sie sie in einem Browser, und Sie sollten die Begrüßungskarte sehen.

</Steps>

<Aside type="note">
Derselbe dreistufige Ablauf funktioniert für jede Demo im Katalog. Wenn Sie eine andere ausprobieren möchten, ersetzen Sie einfach die Demo-ID, zum Beispiel durch `helpdesk-itsm`, `hr-onboarding` oder `weather-mcp-demo`. Die vollständige Liste finden Sie unter [Demos ausführen](/de/getting-started/running-demos/).
</Aside>

## Ihr eigenes Bundle erstellen

Wenn Sie bereit sind, über den veröffentlichten Katalog hinauszugehen, gelten dieselben drei Befehle. Sie verwenden dann einfach Ihre eigenen Dateien statt Release-URLs:

<Steps>

1. **Erstellen Sie ein Bundle** mit dem Wizard. Sie können ihn interaktiv oder mit einem lokalen Answer-Dokument ausführen:

   ```bash
   gtc wizard
   # or
   gtc wizard --answers ./my-create-answers.json
   ```

2. **Konfigurieren Sie Provider**, wieder entweder interaktiv oder nicht interaktiv:

   ```bash
   gtc setup ./my-bundle
   # or
   gtc setup ./my-bundle --answers ./my-setup-answers.json
   ```

3. **Starten Sie die Runtime**:

   ```bash
   gtc start ./my-bundle
   ```

</Steps>

Unter [gtc wizard](/de/cli/wizard/) und [gtc setup](/de/cli/setup/) finden Sie die vollständige Referenz, einschließlich Capability-Flags und Schemas für Answer-Dokumente.

## Beispiel: Hello World Flow

Erstellen Sie einen einfachen Flow, der auf Nachrichten antwortet:

```yaml title="flows/hello.ygtc"
name: hello_world
version: "1.0"
description: A simple greeting flow

nodes:
  - id: greet
    type: reply
    config:
      message: "Hello! I'm your digital worker."

triggers:
  - type: message
    pattern: "hello"
    target: greet
```

## Bundle-Struktur

Ein typisches Greentic-Bundle sieht so aus:

```
my-bundle/
├── greentic.demo.yaml      # Hauptkonfiguration
├── providers/
│   └── messaging/
│       └── messaging-telegram.gtpack
├── apps/
│   └── my-app/
│       └── flows/
│           └── on_message.ygtc
└── seeds.yaml              # Seed-Daten (optional)
```

## Konfigurationsdatei

Die Hauptkonfigurationsdatei (`greentic.demo.yaml`) definiert Ihr Setup:

```yaml title="greentic.demo.yaml"
name: my-digital-worker
version: "1.0"

providers:
  messaging-telegram:
    pack: "providers/messaging/messaging-telegram.gtpack"
    setup_flow: "setup_default"

apps:
  my-app:
    path: "apps/my-app"
    default_flow: "on_message"

tenants:
  demo:
    name: Demo Tenant
    teams:
      default:
        channels:
          telegram:
            provider: messaging-telegram
```
