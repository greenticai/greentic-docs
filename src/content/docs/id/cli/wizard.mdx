## Gambaran Umum

Perintah `gtc wizard` membuat bundle Greentic baru dari jawaban wizard. Perintah ini melakukan scaffold untuk struktur bundle lengkap termasuk file konfigurasi, setup provider, dan template aplikasi.

## Penggunaan

```bash
gtc wizard [OPTIONS]
```

## Opsi

| Opsi | Deskripsi |
|--------|-------------|
| `--answers <FILE \| URL>` | Path atau URL ke file jawaban wizard (JSON atau YAML). Mendukung path lokal dan URL remote `http://` / `https://`. |
| `--dry-run` | Pratinjau file yang dihasilkan tanpa menulis |
| `--output <DIR>` | Direktori output (default: direktori saat ini) |
| `--template <NAME>` | Gunakan template tertentu |
| `-v, --verbose` | Aktifkan output verbose |

## File Jawaban Wizard

### Format JSON

```json title="wizard-answers.json"
{
  "project": {
    "name": "my-digital-worker",
    "version": "1.0.0",
    "description": "Customer support digital worker"
  },
  "providers": {
    "messaging": ["telegram", "slack"],
    "events": ["webhook", "timer"]
  },
  "apps": [
    {
      "name": "support-bot",
      "template": "customer-service"
    }
  ],
  "tenants": [
    {
      "id": "demo",
      "name": "Demo Tenant",
      "teams": [
        {
          "id": "default",
          "channels": ["telegram", "slack"]
        }
      ]
    }
  ]
}
```

### Format YAML

```yaml title="wizard-answers.yaml"
project:
  name: my-digital-worker
  version: "1.0.0"
  description: Customer support digital worker

providers:
  messaging:
    - telegram
    - slack
  events:
    - webhook
    - timer

apps:
  - name: support-bot
    template: customer-service

tenants:
  - id: demo
    name: Demo Tenant
    teams:
      - id: default
        channels:
          - telegram
          - slack
```

### Capability

Array `capabilities` di file jawaban wizard mengontrol fitur level bundle yang diaktifkan saat bundle dibuat. Capability menentukan scaffold tambahan apa yang dijalankan wizard di luar struktur bundle standar.

```json title="create-answers.json"
{
  "capabilities": ["greentic.cap.bundle_assets.read.v1"]
}
```

**Capability yang tersedia:**

| Capability | Deskripsi |
|------------|-------------|
| `greentic.cap.bundle_assets.read.v1` | Mengaktifkan override aset level bundle. Jika ada, wizard akan otomatis melakukan scaffold aset skin webchat-gui dari provider pack ke direktori `assets/` bundle. |

Ketika `greentic.cap.bundle_assets.read.v1` disertakan, bundle yang dihasilkan akan berisi direktori `assets/` dengan file yang disalin dari provider pack (misalnya skin dan konfigurasi webchat-gui). File level bundle ini dapat disesuaikan dan akan menimpa default pack saat runtime.

<Aside type="tip">
Array `capabilities` ditulis ke dalam `bundle.yaml` dan juga dihormati oleh `gtc setup` saat mengonfigurasi bundle nanti. Lihat [bagian capability gtc setup](/id/cli/setup/#capability-bundle) untuk detail perilaku saat runtime.
</Aside>


## Contoh

### Bundle Katalog Demo

Repositori [`greentic-demo`](https://github.com/greenticai/greentic-demo) menerbitkan `*-create-answers.json` dan `*-setup-answers.json` untuk setiap demo dalam artefak release-nya. Gunakan langsung untuk membuat bundle lokal yang masih bisa Anda edit setelahnya:

```bash
gtc wizard --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-create-answers.json
```

Lalu lanjutkan dengan setup dan start:

```bash
gtc setup ./quickstart-demo-bundle --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-setup-answers.json
gtc start ./quickstart-demo-bundle
```

Ganti ID demo dengan salah satu demo yang dipublikasikan (`quickstart`, `quickstart-event`, `hr-onboarding`, `helpdesk-itsm`, `sales-crm`, `supply-chain`, `incident`, `redbutton`, `cloud-deploy-demo`, `weather-mcp-demo`, ...). Katalog lengkap beserta deskripsi dan kredensial yang diperlukan tersedia di halaman [Menjalankan Demo](/id/getting-started/running-demos/).

<Aside type="tip">
Wizard menjalankan **resolusi dependensi berbasis capability**. Ketika dokumen create-answers demo mereferensikan provider yang mendeklarasikan `required_capabilities` (misalnya provider messaging yang membutuhkan `greentic:state/state-store`), wizard otomatis menambahkan pack yang memenuhinya seperti `state-memory` ke bundle yang dihasilkan. Anda tidak perlu mencantumkannya secara manual.
</Aside>

### Penggunaan Dasar

```bash
# Interactive mode
gtc wizard

# With answers file
gtc wizard --answers wizard-answers.yaml
```

### Mode Pratinjau

```bash
# See what would be generated
gtc wizard --answers wizard-answers.yaml --dry-run
```

### File Jawaban Remote

Flag `--answers` menerima URL `http://` dan `https://`, sehingga mudah untuk
menyimpan dokumen jawaban bersama di lokasi terpusat dan mereferensikannya dari pipeline CI/CD.

```bash
# Fetch answers from a remote URL
gtc wizard --answers https://config.example.com/teams/support/wizard-answers.yaml

# Combine with dry-run to preview before generating
gtc wizard --answers https://config.example.com/teams/support/wizard-answers.yaml --dry-run
```

<Aside type="tip">
Dukungan URL remote berguna di pipeline CI/CD ketika jawaban wizard disimpan di repositori bersama
atau artifact server dan diambil saat build berlangsung.
</Aside>

### Direktori Output Kustom

```bash
gtc wizard --answers wizard-answers.yaml --output ./my-project
```

```bash
gtc wizard --list-templates

gtc wizard --template customer-service --answers answers.yaml
```

## Capability Bundle

Wizard akan menanyakan capability bundle opsional yang mengaktifkan fitur tambahan. Setiap capability membuka kumpulan aset dan perilaku tertentu di bundle yang dihasilkan.

### Prompt Interaktif

Saat menjalankan `gtc wizard` secara interaktif, Anda akan melihat:

```
? Enable bundle-level assets (./assets/)? [Y/n]
? Enable OAuth login for WebChat GUI? [y/N]
? Enable multi-language for WebChat GUI? [y/N]
? Enable embeddable WebChat widget? [y/N]
```

### Referensi Capability

| Prompt | ID Capability | Fungsinya |
|--------|---------------|-------------|
| Enable bundle-level assets | `greentic.cap.bundle_assets.read.v1` | Membuat direktori `./assets/` dan melakukan scaffold file skin default dari provider pack ke direktori per-tenant |
| Enable OAuth login for WebChat GUI | `greentic.cap.webchat.oauth.v1` | Menambahkan konfigurasi auth provider OAuth ke file konfigurasi tenant |
| Enable multi-language for WebChat GUI | `greentic.cap.webchat.i18n.v1` | Mengaktifkan locale picker di WebChat dan melakukan scaffold file JSON locale di bawah `./assets/` |
| Enable embeddable WebChat widget | `greentic.cap.webchat.embed.v1` | Menghasilkan embed HTML snippet (fullpage link, inline iframe, chat bubble) per tenant |

### Mengatur Capability di File Jawaban

Alih-alih menjawab prompt secara interaktif, Anda bisa mengatur capability langsung di file jawaban dengan menambahkan array `capabilities`:

```json title="wizard-answers.json"
  "tenants": [

      "id": "demo",

      "teams": [{ "id": "default", "channels": ["webchat"] }]
    }
  ]
}
```
  </TabItem>

```yaml title="wizard-answers.yaml"
  name: my-digital-worker

    - webchat
    - telegram

  - greentic.cap.bundle_assets.read.v1
  - greentic.cap.webchat.oauth.v1
```

<Aside type="tip">
Capability bersifat aditif. Mengaktifkan `greentic.cap.webchat.embed.v1` secara implisit memerlukan `greentic.cap.bundle_assets.read.v1` karena embed snippet ditulis ke direktori `./assets/`. Wizard menangani dependensi ini secara otomatis.
</Aside>

Untuk detail lengkap tentang capability, perilaku scaffold, dan mekanisme overlay, lihat [Bundle Assets](/id/greentic-docs/concepts/bundle-assets/).

## Alur Kerja

<Steps>

1. **Buat file jawaban**

   Definisikan konfigurasi proyek Anda dalam JSON atau YAML.

2. **Jalankan wizard**

   ```bash
   gtc wizard --answers wizard-answers.yaml
   ```

3. **Tinjau file yang dihasilkan**

   Periksa struktur bundle yang dihasilkan.

4. **Konfigurasikan provider**

   ```bash
   gtc setup ./my-digital-worker
   ```

5. **Mulai runtime**

   ```bash
   gtc start ./my-digital-worker
   ```

</Steps>

## Menyesuaikan Flow yang Dihasilkan

Wizard menghasilkan template flow dasar. Sesuaikan flow tersebut setelah generation:

```yaml title="apps/support-bot/flows/on_message.ygtc"
name: on_message
version: "1.0"
description: Handle incoming messages

nodes:
  # Add your custom nodes here
  - id: analyze
    type: llm
    config:
      model: "gpt-4"
      prompt: "Analyze: {{message}}"
    next: respond

  - id: respond
    type: reply
    config:
      message: "{{analysis_result}}"

triggers:
  - type: message
    target: analyze
```

<Aside type="tip">
Gunakan `--dry-run` terlebih dahulu untuk meninjau struktur yang akan dihasilkan sebelum menulisnya ke disk.
</Aside>

## Component Scaffolding: `http_client` Capability

When scaffolding a new WASM component (for example via `greentic-component new`), the wizard can mark the component as needing outbound HTTP by setting `http_client: true`. This writes the capability declaration into the component manifest.

<Aside type="caution">
**Desktop runner HTTP gate.** Declaring `http_client: true` is a capability _request_ — the runtime must grant it before the component can actually open outbound sockets. On the **desktop runner profile**, honouring this manifest flag has historically been a no-op: components that declared `http_client: true` had no `wasi:http/outgoing-handler` implementation linked, and runtime HTTP calls failed.

Starting with Greentic runner `N.N.N` (landing with the desktop HTTP gate fix, not yet released), the desktop profile will honour the manifest and grant `wasi:http/outgoing-handler@0.2.0` automatically when a component declares `http_client: true`. Until that release is available, use the workaround below. {/* TODO: confirm runner version once the desktop HTTP gate fix lands */}

**Workaround until the fix ships:** ship your component with a custom WIT world that imports `wasi:http/outgoing-handler@0.2.0` directly, rather than relying on the scaffolded manifest flag. Once you are on a runner that includes the desktop HTTP gate manifest-honouring fix, you can drop the custom world and use `http_client: true` as intended.

For components that emit Adaptive Cards via HTTP fetches (e.g. RAG lookups before returning an `attachments[]` array), see the [WebChat provider's Attachment Envelope Shape](/id/providers/messaging/webchat/#attachment-envelope-shape) section for the expected component-output layout.
</Aside>

## Pemecahan Masalah

### File Jawaban Tidak Valid

```
Error: Failed to parse answers file
```

Pastikan JSON/YAML Anda valid. Gunakan linter untuk memeriksa sintaks.

### Field Wajib Tidak Ada

```
Error: Missing required field: project.name
```

Periksa bahwa semua field yang diperlukan ada di file jawaban Anda.

### Template Tidak Ditemukan

```
Error: Template 'unknown' not found
```

Gunakan `gtc wizard --list-templates` untuk melihat template yang tersedia.
