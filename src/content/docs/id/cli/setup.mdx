## Gambaran Umum

**Greentic Setup** adalah mesin konfigurasi untuk bundle Greentic. Menjalankan `gtc setup <bundle>` akan membuka dasbor web interaktif di browser Anda untuk mengonfigurasi kredensial provider, memeriksa scope saat ini, dan menjalankan wizard pengaturan terpandu untuk setiap provider — semuanya dari aplikasi satu halaman yang dihosting secara lokal dan dilindungi.

Untuk lingkungan CI atau SSH, `gtc setup --no-ui <bundle>` akan beralih ke alur dialoguer terminal.

## Menjalankan

```bash
# Luncurkan dasbor web (default)
gtc setup ./my-bundle

# Mode terminal tanpa antarmuka (CI / SSH)
gtc setup --no-ui ./my-bundle

# Terapkan file jawaban yang sudah disiapkan tanpa UI
gtc setup --answers answers.json ./my-bundle
```

Saat Anda menjalankan `gtc setup ./my-bundle`, binary akan:

1. Memuat dan memvalidasi konfigurasi bundle Anda.
2. Memulai server HTTP [Axum](https://docs.rs/axum) lokal yang terikat ke `127.0.0.1` pada port acak yang tersedia.
3. Menghasilkan token bearer sekali pakai yang hanya disimpan di memori — tidak pernah ditulis ke disk.
4. Membuka browser default Anda ke `http://127.0.0.1:<port>/?token=<token>`.
5. Melayani aplikasi satu halaman Greentic Setup.

Server akan otomatis mati saat Anda menutup tab browser atau menggunakan tombol **Close** di dasbor.

## Dasbor

Dasbor Greentic Setup adalah aplikasi satu halaman yang dibangun dengan [Alpine.js](https://alpinejs.dev/). Ini menyediakan antarmuka yang bersih dan minimalis untuk mengelola konfigurasi bundle Anda tanpa meninggalkan browser.

### Tata Letak

Dasbor memiliki tiga area utama:

- **Topbar** — menampilkan nama bundle dan tombol tutup yang mematikan server dengan benar.
- **Sidebar** — berisi tiga pemilih scope (tenant, lingkungan, tim) dan menu navigasi.
- **Area konten utama** — merender tampilan yang aktif.

### Navigasi sidebar

Tiga pemilih dropdown di bagian atas sidebar memungkinkan Anda mengubah konteks scope yang aktif:

| Pemilih | Yang dikontrol |
|---------|----------------|
| Tenant | Namespace tenant untuk rahasia dan konfigurasi provider |
| Lingkungan | `demo`, `dev`, `staging`, atau nilai kustom |
| Tim | Scope sub-tim dalam tenant |

Bagian navigasi **Manage** menampilkan:

| Item | Status |
|------|--------|
| Overview | Aktif di Phase 1a |
| Providers | Akan hadir di Phase 1b |
| Secrets | Akan hadir di Phase 1b |
| Capabilities | Akan hadir di Phase 1b |

Item yang ditandai "1b" adalah placeholder hanya-baca di rilis saat ini.

### Tampilan Overview

Overview adalah tampilan landing default. Berisi:

- **Judul selamat datang** dengan nama tampilan bundle.
- **Baris statistik** yang menampilkan empat chip ringkasan: Scope yang dikonfigurasi, Provider yang ditemukan, Rahasia yang tersimpan, dan Peringatan.
- **Daftar scope** — satu kartu per scope yang dikonfigurasi (kombinasi tenant/lingkungan/tim), masing-masing menampilkan chip provider yang telah diatur di scope tersebut.
- **Status kosong** — jika belum ada scope yang dikonfigurasi, panel status kosong akan muncul dan mengajak Anda menambahkan tenant pertama menggunakan tombol **Add tenant**.

## Wizard

Mengklik **Add tenant** (atau tombol **Configure** per scope) akan meluncurkan wizard pengaturan inline. Wizard berjalan di dalam shell dasbor — tidak berpindah ke halaman lain.

### Alur wizard

Wizard mengumpulkan konfigurasi dalam serangkaian langkah:

<Steps>

1. **Pilih scope** — pilih atau konfirmasi tenant, lingkungan, dan tim.

2. **Pilih provider** — aktifkan provider mana yang ingin Anda aktifkan untuk scope ini.

3. **Konfigurasi setiap provider** — untuk setiap provider yang diaktifkan, isi field yang diperlukan (kunci API, URL webhook publik, dll.).

4. **Tinjau** — lihat ringkasan semua nilai sebelum disimpan.

5. **Selesai** — kredensial ditulis ke penyimpanan rahasia; Overview diperbarui untuk menampilkan scope baru.

</Steps>

### Jenis field

Wizard mendukung jenis input berikut:

| Jenis | Penggunaan |
|-------|-----------|
| `text` | Nilai string biasa (URL, ID, nama pengguna) |
| `password` | Nilai sensitif — dirender sebagai input tersamar, dihapus dari memori setelah selesai |
| `textarea` | Nilai multi-baris (blok PEM sertifikat, payload JSON) |
| `select` | Pilihan yang ditetapkan |
| `switch` | Toggle boolean (aktifkan/nonaktifkan provider) |

### Navigasi

Setiap langkah memiliki aksi **Back**, **Continue**, dan **Cancel**. Langkah terakhir mengganti **Continue** dengan **Finish**. Validasi berjalan per langkah sebelum melanjutkan; kesalahan ditampilkan secara inline di bawah field yang relevan.

## Mode tanpa antarmuka

Jika Anda bekerja melalui SSH, dalam pipeline CI, atau telah menonaktifkan peluncuran browser, gunakan `--no-ui`:

```bash
gtc setup --no-ui ./my-bundle
```

Ini memanggil alur dialoguer terminal warisan. Semua nilai konfigurasi yang sama dikumpulkan secara interaktif di prompt, dan rahasia disimpan ke penyimpanan yang sama dengan wizard dasbor.

### File jawaban

File jawaban memungkinkan Anda mengotomatiskan pengaturan tanpa prompt interaktif apa pun:

```bash
# Hasilkan template (hanya field yang diperlukan)
gtc setup --dry-run --emit-answers answers.json ./my-bundle

# Hasilkan template dengan field opsional
gtc setup --dry-run --emit-answers answers.json --advanced ./my-bundle

# Terapkan jawaban
gtc setup --answers answers.json ./my-bundle
```

#### Jawaban katalog demo

Setiap demo dalam katalog [`greentic-demo`](https://github.com/greenticai/greentic-demo) menerbitkan dokumen `*-setup-answers.json`. Terapkan langsung:

```bash
gtc setup ./quickstart-demo-bundle \
  --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-setup-answers.json
```

Lihat [Menjalankan Demo](/id/getting-started/running-demos/) untuk katalog lengkap dengan deskripsi dan kredensial yang diperlukan.

<Aside type="note">
Greentic Setup secara otomatis mendeteksi tenant bundle dari direktori `tenants/`, sehingga rahasia ditulis ke URI tenant yang benar bahkan ketika bundle tidak dibuat dengan tenant `demo` default. Ia juga menjalankan **validasi kemampuan dependensi** di setiap pack dalam bundle dan memperingatkan jika ada kemampuan yang diperlukan (misalnya `greentic:state/state-store`) yang belum terpenuhi.
</Aside>

## Keamanan

Greentic Setup mengutamakan keamanan untuk semua sesi UI lokal:

| Kontrol | Implementasi |
|---------|-------------|
| Hanya loopback | Server hanya terikat ke `127.0.0.1` — tidak ada eksposur jaringan |
| Port acak | Port acak yang tersedia dipilih saat startup — tidak ada port tetap yang bisa ditebak |
| Token bearer | Token sekali pakai yang dihasilkan di memori, diteruskan melalui parameter query URL pada pemuatan pertama, kemudian hanya disimpan di sesi browser |
| Penerapan Origin | Setiap permintaan API divalidasi terhadap daftar izin `Origin`; permintaan lintas-origin ditolak |
| Tidak ada rahasia di log | Nilai kredensial tidak pernah ditulis ke stdout, stderr, atau file log |
| Tidak ada rahasia di disk | Token bearer dan jawaban wizard hanya disimpan di memori; `Zeroize` dipanggil pada struct sensitif setelah digunakan |
| Aset asal yang sama | Semua aset JS, CSS, dan font dilayani dari server lokal yang sama — tidak ada panggilan CDN eksternal |
| Tidak ada telemetri | Dasbor tidak melakukan panggilan jaringan keluar |

## Kemampuan bundle

File `bundle.yaml` dapat mendeklarasikan kemampuan yang membuka perilaku tambahan selama pengaturan.

### Aset bundle (`greentic.cap.bundle_assets.read.v1`)

Ketika kemampuan ini dideklarasikan, Greentic Setup membuat direktori `assets/` di root bundle dan menyalin aset pack provider ke dalamnya (misalnya `assets/webchat-gui/`). Anda kemudian dapat memodifikasi file-file ini dengan bebas; file aset tingkat bundle menggantikan default pack saat runtime.

```yaml title="bundle.yaml"
capabilities:
  - greentic.cap.bundle_assets.read.v1
```

<Aside type="tip">
Setelah pengaturan membuat aset, kustomisasi tingkat bundle Anda mengambil prioritas di atas default pack ketika runtime memuatnya.
</Aside>

## Peta jalan

Greentic Setup Phase 1a menghadirkan shell dasbor, tampilan Overview, dan wizard inline. Pekerjaan mendatang di **Phase 1b** mencakup:

- **Providers** — telusuri semua pack dalam bundle, aktifkan atau nonaktifkan provider individual, jalankan ulang pengaturan untuk satu provider tanpa menyentuh yang lain.
- **Secrets** — CRUD lengkap untuk penyimpanan rahasia: buat, baca (bertopeng), perbarui, dan hapus entri rahasia individual per scope.
- **Capabilities** — aktifkan flag kemampuan per pack dan periksa grafik dependensi.
- **Pemicu rebuild** — minta hot reload bundle yang berjalan dari dasbor tanpa memulai ulang runtime penuh.

## Pemecahan masalah

### Browser tidak terbuka

Jika sistem Anda tidak mendukung peluncuran browser otomatis, salin URL yang dicetak ke terminal dan tempelkan secara manual ke browser Anda:

```
Greentic Setup running at http://127.0.0.1:52341/?token=...
```

### Pendaftaran webhook gagal

```
Error: Failed to register webhook: 401 Unauthorized
```

Periksa bahwa token bot Anda benar dan belum kedaluwarsa.

### Field yang diperlukan tidak ada

```
Error: Missing required field 'bot_token' for messaging-telegram
```

Pastikan semua field yang diperlukan diisi dalam wizard atau ada dalam file jawaban Anda.

### URL publik tidak dapat diakses

```
Error: Webhook verification failed: Connection refused
```

URL publik Anda harus dapat dijangkau dari internet. Periksa bahwa ngrok atau cloudflared sedang berjalan.

### Kredensial tidak ditemukan

```
Error: Secret 'slack_bot_token' not found
```

Jalankan `gtc setup` untuk mengonfigurasi provider, atau periksa file jawaban Anda.
