## Ringkasan

Provider messaging Email memungkinkan digital worker Anda berkomunikasi melalui email. Provider ini mendukung:

- Mengirim email
- Menerima email (melalui webhook)
- Pemformatan HTML
- Lampiran

## Penyiapan

<Steps>

1. **Konfigurasikan Pengaturan SMTP**

   ```json title="answers.json"
   {
     "messaging-email": {
       "enabled": true,
       "smtp_host": "smtp.example.com",
       "smtp_port": 587,
       "smtp_user": "your-email@example.com",
       "smtp_password": "your-password",
       "from_address": "support@example.com",
       "from_name": "Support Bot"
     }
   }
   ```

2. **Konfigurasikan Inbound (Opsional)**

   Untuk menerima email, siapkan webhook dengan provider email Anda:
   - SendGrid Inbound Parse
   - Mailgun Routes
   - AWS SES

3. **Jalankan Setup**

   ```bash
   gtc setup --answers answers.json ./my-bundle
   ```

</Steps>

## Opsi Konfigurasi

| Opsi | Wajib | Deskripsi |
|------|-------|-----------|
| `enabled` | Ya | Aktifkan/nonaktifkan provider |
| `smtp_host` | Ya | Hostname server SMTP |
| `smtp_port` | Ya | Port SMTP (biasanya 587 atau 465) |
| `smtp_user` | Ya | Username SMTP |
| `smtp_password` | Ya | Password SMTP |
| `from_address` | Ya | Alamat pengirim default |
| `from_name` | Tidak | Nama tampilan untuk alamat pengirim |
| `webhook_url` | Tidak | Endpoint webhook inbound |

## Fitur

### Kirim Plain Text

```yaml
- id: send_email
  type: reply
  config:
    to: "{{user_email}}"
    subject: "Your Support Request"
    message: "Thank you for contacting us. We'll get back to you shortly."
```

### Kirim HTML

```yaml
- id: send_html_email
  type: reply
  config:
    to: "{{user_email}}"
    subject: "Welcome!"
    html: |
      <html>
        <body>
          <h1>Welcome to Our Service!</h1>
          <p>Thank you for signing up.</p>
          <a href="https://example.com/start">Get Started</a>
        </body>
      </html>
```

### Dengan Lampiran

```yaml
- id: send_with_attachment
  type: reply
  config:
    to: "{{user_email}}"
    subject: "Your Report"
    message: "Please find your report attached."
    attachments:
      - url: "https://example.com/report.pdf"
        filename: "report.pdf"
```

### CC dan BCC

```yaml
- id: send_copy
  type: reply
  config:
    to: "{{user_email}}"
    cc: "manager@example.com"
    bcc: "archive@example.com"
    subject: "Support Update"
    message: "Your ticket has been resolved."
```

## Menangani Email Masuk

```yaml title="flows/on_email.ygtc"
name: handle_email
version: "1.0"

nodes:
  - id: process
    type: reply
    config:
      to: "{{from_address}}"
      subject: "Re: {{subject}}"
      message: "Thank you for your email. We've received your message."

triggers:
  - type: email
    target: process
```
