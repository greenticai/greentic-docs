Panduan ini membawa Anda dari nol hingga menjalankan digital worker Greentic dalam beberapa menit. Jalur tercepat adalah menjalankan salah satu demo yang dipublikasikan dari katalog [`greentic-demo`](https://github.com/greenticai/greentic-demo). Setiap demo menggunakan alur tiga langkah yang sama.

<Aside type="tip">
Ingin menelusuri semua demo yang tersedia dan melihat dengan tepat apa yang dilakukan masing-masing? Langsung ke [Menjalankan Demo](/id/getting-started/running-demos/).
</Aside>

## Prasyarat

Sebelum memulai, pastikan Anda memiliki:

- **`gtc` CLI** terinstal — lihat [Instalasi](/id/getting-started/installation/)
- **Rust** 1.91 atau lebih baru (hanya diperlukan jika Anda build dari source)
- **URL publik** untuk provider webhook (misalnya `ngrok` atau `cloudflared`) — tidak diperlukan untuk quickstart yang hanya memakai WebChat

## Jalankan demo quickstart

Demo `quickstart` adalah bundle end-to-end paling kecil: kartu sambutan, kartu tentang, dan interaksi echo melalui WebChat. Demo ini tidak memerlukan akun eksternal.

<Steps>

1. **Buat bundle** dari dokumen create-answers yang dipublikasikan:

   ```bash
   gtc wizard --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-create-answers.json
   ```

   Wizard mengunduh demo pack, menyelesaikan dependensi capability (misalnya menarik `state-memory` secara otomatis karena provider WebChat memerlukan `greentic:state/state-store`), lalu menulis direktori bundle lokal.

2. **Konfigurasikan provider** dengan dokumen setup-answers yang dipublikasikan:

   ```bash
   gtc setup ./quickstart-demo-bundle --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-setup-answers.json
   ```

   Setup mendeteksi tenant bundle secara otomatis, menyimpan secret per provider, dan memvalidasi bahwa setiap capability yang dibutuhkan sudah terpenuhi.

3. **Mulai runtime**:

   ```bash
   gtc start ./quickstart-demo-bundle
   ```

   Runner mencetak URL publik WebChat. Buka URL tersebut di browser dan Anda seharusnya melihat kartu sambutan.

</Steps>

<Aside type="note">
Alur tiga langkah yang sama berlaku untuk setiap demo di katalog. Untuk mencoba demo lain, cukup ganti ID demonya, misalnya `helpdesk-itsm`, `hr-onboarding`, atau `weather-mcp-demo`. Lihat [Menjalankan Demo](/id/getting-started/running-demos/) untuk daftar lengkapnya.
</Aside>

## Membangun bundle Anda sendiri

Saat Anda siap melangkah lebih jauh dari katalog yang dipublikasikan, tiga perintah yang sama tetap berlaku. Bedanya, perintah tersebut menggunakan file Anda sendiri, bukan URL release:

<Steps>

1. **Buat bundle** dengan wizard. Anda dapat menjalankannya secara interaktif atau dengan dokumen jawaban lokal:

   ```bash
   gtc wizard
   # or
   gtc wizard --answers ./my-create-answers.json
   ```

2. **Konfigurasikan provider**, sekali lagi baik secara interaktif maupun non-interaktif:

   ```bash
   gtc setup ./my-bundle
   # or
   gtc setup ./my-bundle --answers ./my-setup-answers.json
   ```

3. **Mulai runtime**:

   ```bash
   gtc start ./my-bundle
   ```

</Steps>

Lihat [gtc wizard](/id/cli/wizard/) dan [gtc setup](/id/cli/setup/) untuk referensi lengkap, termasuk flag capability dan skema dokumen answers.

## Contoh: Hello World Flow

Buat flow sederhana yang merespons pesan:

```yaml title="flows/hello.ygtc"
name: hello_world
version: "1.0"
description: A simple greeting flow

nodes:
  - id: greet
    type: reply
    config:
      message: "Hello! I'm your digital worker."

triggers:
  - type: message
    pattern: "hello"
    target: greet
```

## Struktur Bundle

Bundle Greentic yang umum terlihat seperti ini:

```
my-bundle/
├── greentic.demo.yaml      # Main configuration
├── providers/
│   └── messaging/
│       └── messaging-telegram.gtpack
├── apps/
│   └── my-app/
│       └── flows/
│           └── on_message.ygtc
└── seeds.yaml              # Seed data (optional)
```

## File Konfigurasi

File konfigurasi utama (`greentic.demo.yaml`) mendefinisikan setup Anda:

```yaml title="greentic.demo.yaml"
name: my-digital-worker
version: "1.0"

providers:
  messaging-telegram:
    pack: "providers/messaging/messaging-telegram.gtpack"
    setup_flow: "setup_default"

apps:
  my-app:
    path: "apps/my-app"
    default_flow: "on_message"

tenants:
  demo:
    name: Demo Tenant
    teams:
      default:
        channels:
          telegram:
            provider: messaging-telegram
```
