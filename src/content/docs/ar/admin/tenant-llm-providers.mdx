# موفّرو LLM لكل مستأجر

تعمل ميزات الذكاء الاصطناعي في Designer — منشئ التدفقات، ومؤلِّف Agentic Worker،
ومؤلِّفا SoRLa وOperaLa — جميعها عبر نموذج لغوي كبير (LLM). افتراضيًا
يكوّن مشغل المنصة هؤلاء الموفّرين مركزيًا. ومع
**موفّري LLM لكل مستأجر**، يمكن للمستأجر أيضًا إحضار مفتاح واجهة برمجة التطبيقات الخاص به وجعل
Designer يستخدمه.

## مصدران، قاعدة واحدة

لصف الموفّر **مصدر**:

- **operator** — يحدده مشغل المنصة، للقراءة فقط بالنسبة للمستأجر.
- **tenant** — يُنشئه ويملكه مسؤول المستأجر.

قاعدة الحل بسيطة وقابلة للتنبؤ: **مفتاح المشغل يفوز
دائمًا**. لا يمكن لمفتاح المستأجر إلا أن يملأ دور مؤلِّف *لم* يكن المشغل قد
عيّنه بالفعل. لا يمكن للمستأجر أبدًا الاستيلاء على (أو "سرقة") دور
يديره المشغل.

<Aside type="note">
هذا يحافظ على سلامة الحوكمة المركزية. إذا فرضت منصتك موفّرًا
معيّنًا، على سبيل المثال، لمؤلِّف Agentic Worker، فإن تعيينه كدور مشغل
يضمن أن يستخدمه كل مستأجر — ومع ذلك يمكن لمفتاح المستأجر الخاص أن يخدم
الأدوار التي تركتها مفتوحة.
</Aside>

## أدوار المؤلِّف

يمكن تعيين كل موفّر LLM لواحد أو أكثر من **أدوار المؤلِّف**، التي تتوافق
مع واجهات Designer التي تستخدمها:

| الدور | يشغّل |
| --- | --- |
| `flow_editor_composer` | منشئ التدفقات (BUILD / EDIT) ودردشة الذكاء الاصطناعي |
| `agentic_worker_composer` | مؤلِّف Agentic Worker / DW |
| `sorla_composer` | استوديو تأليف SoRLa |
| `operala_composer` | لوحة عمليات OperaLa |
| `telco_playbook_composer` | تأليف دليل عمليات الاتصالات |

يملك الدور موفّر واحد بالضبط. يُرفض تعيين دور يملكه المشغل
بالفعل، بحيث لا يوجد أبدًا سؤال غامض عن "أي مفتاح يخدم هذه
الواجهة".

## تدفق مسؤول المستأجر

في Designer، يفتح مسؤول المستأجر **Settings → Credentials → My LLM
providers**. ومن هناك يمكنه:

- إضافة موفّر (OpenAI، Anthropic، Azure OpenAI، Google، وغيرها) بمفتاح
  واجهة برمجة التطبيقات الخاص به، وعنوان URL أساسي اختياري، ونموذج،
- اختبار المفتاح قبل الحفظ،
- تعيينه لأي دور مؤلِّف لم يحجزه المشغل،
- إزالة موفّر يملكه.

تُعرض الأدوار المحجوزة من قبل المشغل لكنها معطّلة — إشارة واضحة لما
يُدار مركزيًا مقابل ما يتحكم فيه المستأجر.

## كيفية التعامل مع المفاتيح

- تُشفَّر مفاتيح المستأجر أثناء التخزين وتُفك تشفيرها فقط في الذاكرة عندما يحتاج
  أحد الطلبات إليها.
- يحل Designer مكدسات LLM الخاصة به من الإدارة ويُحدّثها خلال
  حوالي 30 ثانية من أي تغيير، بحيث تسري إضافة مفتاح أو تدويره دون
  إعادة تشغيل.
- يعود الدور غير المعيَّن إلى تكوين بدء التشغيل المحلي لـ Designer
  عند توفيره؛ ومع عدم وجود مفتاح لدور ما، تُبلغ تلك الواجهة أنه لم يُكوَّن أي
  موفّر بدلاً من استخدام الموفّر الخاطئ بصمت.
