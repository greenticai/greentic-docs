# スターターテンプレート

**スターターテンプレート**とは、ユーザーが Designer で新しいプロジェクトを作成するときに選べる、精査された出発点のことです。白紙のキャンバスの代わりに、組織の慣習をすでに反映したフローから始められます。

運用者はテナント — または特定のチーム — に表示されるテンプレートをキュレーションするため、ユーザーが開くギャラリーは常にそのユーザーにとって適切なものになります。

## テンプレートのキュレーション(運用者)

管理コンソールで、プラットフォーム運用者はテナント向けに表示されるテンプレートを選択します。キュレーションはユーザーごとに解決されます。ユーザーが見るギャラリーは、テナント全体のテンプレートと、そのユーザーが所属するチーム向けにキュレーションされたテンプレートの和集合です。

<Aside type="note">
解決されるセットはユーザーのチームメンバーシップに依存するため、同じテナント内の 2 人のユーザーが異なるスターターギャラリーを見ることがあります。各ユーザーは、自分のチーム向けにキュレーションされたテンプレートに加えてテナント全体のものを、ちょうど見ることになります。
</Aside>

## テンプレートから始める(ユーザー)

Designer で、ユーザーは **New project** ダイアログを開いて **Start from a template** を選びます。ピッカーにはキュレーションされたスターターが一覧表示され、1 つを選択すると、新しいプロジェクトにそのテンプレートのフローが事前入力され、編集できる状態になります。

白紙のオプションを選んでも従来どおりそのまま機能します。テンプレートは任意の助走であり、要件ではありません。

## 良いスターターの条件

- 明確で説明的な名前と概要。ユーザーが開かなくても適切なものを選べるようにします。
- 実行できるほど完成しているが、適応できるほど汎用的なフロー。
- チームが従うべき慣習 — 命名、ノード構造、チャネルの配線 — が組み込まれており、テンプレートが加速させるだけでなく教育にもなるようにします。
