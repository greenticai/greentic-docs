# クラウドデプロイ環境

**デプロイ環境**とは、プラットフォーム運用者がテナント向けに設定する、名前付きのクラウド宛先 — Greentic Cloud、AWS、Azure、または GCP — のことです。環境が作成されると、テナント管理者はクラウド認証情報やデプロイヤー CLI に一切触れることなく、公開済みのパックを Designer から直接その環境にデプロイできます。

このページでは、それを成り立たせる 3 つの要素について説明します。

1. **環境** — 運用者がテナントごと(およびオプションでチームごと)に設定する名前付きの宛先。
2. **デプロイプラグイン** — テナントが使用を許可されているデプロイターゲット。
3. **Designer からのクラウドデプロイ** — テナント管理者によるセルフサービスフロー。

## 役割の概要

| アクション | 担当 |
| --- | --- |
| 環境の作成 / 編集 / 削除 | プラットフォーム運用者のみ |
| 持ち込みクラウド(BYOC)認証情報の設定 | プラットフォーム運用者のみ |
| テナントのデプロイターゲットの有効化 / 無効化 | プラットフォーム運用者のみ |
| 環境の一覧表示とパックのデプロイ | テナント管理者(Designer 内) |

この分担は意図的なものです。運用者はクラウドアカウントと認証情報を所有し、テナントは*何を*、*いつ*出荷するかの判断を所有します。

## 環境

管理コンソールでテナントを開き、**Environments** タブに移動します。各環境には次の項目があります。

- **名前**(テナント内で一意)
- **ターゲット** — `greentic_cloud`、`aws`、`azure`、または `gcp`
- 任意の**リージョン**
- 任意の**スコープ** — テナント全体(すべてのチームに表示)または単一のチームに固定
- **ステータス** — `idle`、`provisioning`、`live`、または `error`

### Greentic Cloud と持ち込みクラウド

- **Greentic Cloud**(`greentic_cloud`)はプラットフォーム自身のマネージドロールでデプロイします。テナントごとの認証情報は不要で、環境を作成した時点ですぐに動作します。
- **AWS / Azure / GCP** は*持ち込みクラウド*(BYOC)です。テナントがこれらの環境にデプロイできるようになる前に、運用者がその環境のクラウド認証情報を添付する必要があります。それまでは、デプロイの試行は「先に認証情報を設定してください」という明確なメッセージとともに拒否されます。

<Aside type="note">
BYOC 認証情報は環境上のシークレット参照として保存され、Designer に返されたりテナントユーザーに表示されたりすることは決してありません。Designer は認証情報が存在するかどうかだけを知り、その値を知ることはありません。
</Aside>

### チームごとの環境

環境は、テナント全体ではなく単一のチームにスコープを限定できます。チームスコープの環境はそのチームのメンバーにのみ表示され、その BYOC 認証情報は所有チームのシークレットスコープに分離されます。1 つの環境をすべてのチームで共有するには、スコープを*テナント全体*のままにします。

## デプロイプラグイン

すべてのテナントがすべてのクラウドに出荷できるべきではありません。**デプロイプラグイン**を使うと、運用者は個々のデプロイターゲットをテナントごと、チームごとにオン / オフできます。

**Environments** タブのデプロイターゲットのトグルは、環境を作成する際にどのターゲットが表示されるかを制御します。利用可能なターゲットのカタログはデプロイヤー自体から取得されるため(実際に実行可能なターゲットのみが提供されます — `greentic_cloud`、`aws`、`azure`、`gcp`)、一覧は常にプラットフォームがデプロイできる内容と一致します。

解決順序は**チーム → テナント → デフォルト有効**です。チームレベルの設定がテナントレベルの設定に優先し、未設定のものはデフォルトで有効になります。無効化されたターゲット向けの環境作成はブロックされます。

## Designer からのクラウドデプロイ

運用者が少なくとも 1 つの環境を設定すると、テナント管理者は Designer からデプロイできます。

1. **Settings → Credentials** を開きます。
2. **Cloud deploy** をクリックします。
3. 出荷する**パック参照**を入力します(例: `my-pack@1.0.0`)。
4. ターゲット環境を選択し、**Deploy** をクリックします。

デプロイがキューに入れられ、環境は `provisioning` に移行します。選択した環境が、認証情報がまだ設定されていない BYOC ターゲットの場合、デプロイは静かに失敗するのではなく、運用者向けの理由とともに拒否されます。

<Aside type="tip">
**Cloud deploy** ボタンはテナント管理者にのみ表示されます。通常のメンバーには表示されず、基盤となる API もそれらの呼び出しを拒否します。
</Aside>

### スタンドアロン Designer

Designer が管理サーバーなしで実行されている場合(スタンドアロンモード)、クラウドデプロイは単に提供されません。環境一覧は空になり、パネルには管理サーバーが設定されていないことが説明されます。デプロイウィザードを通じたローカルおよびバンドルベースのデプロイは影響を受けません。
