このガイドでは、ゼロの状態から数分で Greentic digital worker を動かせるところまで進めます。最短ルートは、[`greentic-demo`](https://github.com/greenticai/greentic-demo) catalog から公開済み demo の 1 つを起動することです。どの demo も同じ 3 ステップの flow を使います。

<Aside type="tip">
利用可能な demo をすべて見て、それぞれが何をするのか正確に確認したい場合は、[Running Demos](/ja/getting-started/running-demos/) に進んでください。
</Aside>

## Prerequisites

開始する前に、以下を確認してください:

- **`gtc` CLI** がインストール済みであること。詳細は [Installation](/ja/getting-started/installation/)
- **Rust** 1.91 以降（source から build する場合のみ必要）
- webhook provider 用の **public URL**（例: `ngrok` または `cloudflared`）。WebChat-only quickstart では不要です

## quickstart demo を起動する

`quickstart` demo は最小の end-to-end bundle です。welcome card、about card、WebChat 経由の echo interaction が含まれます。外部アカウントは不要です。

<Steps>

1. **bundle を作成する**。公開されている create-answers document を使います:

   ```bash
   gtc wizard --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-create-answers.json
   ```

   wizard は demo pack をダウンロードし、capability dependencies を解決し（例: WebChat provider が `greentic:state/state-store` を必要とするため `state-memory` を自動的に取り込みます）、ローカル bundle directory を書き出します。

2. **provider を設定する**。公開されている setup-answers document を使います:

   ```bash
   gtc setup ./quickstart-demo-bundle --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-setup-answers.json
   ```

   setup は bundle の tenant を自動検出し、provider ごとに secret を保存し、必要な capability がすべて満たされていることを検証します。

3. **runtime を起動する**:

   ```bash
   gtc start ./quickstart-demo-bundle
   ```

   runner が public WebChat URL を表示します。browser で開くと welcome card が表示されるはずです。

</Steps>

<Aside type="note">
catalog 内のすべての demo で同じ 3 ステップの flow が使えます。別の demo を試すには、demo ID を置き換えるだけです。たとえば `helpdesk-itsm`、`hr-onboarding`、`weather-mcp-demo` などです。完全な一覧は [Running Demos](/ja/getting-started/running-demos/) を参照してください。
</Aside>

## 独自の bundle を作成する

公開 catalog を超えて進む準備ができたら、同じ 3 つのコマンドを使えます。違いは、release URLs の代わりに自分の file を使うことだけです:

<Steps>

1. **wizard で bundle を作成する**。対話モードでも、ローカル answers document を使っても構いません:

   ```bash
   gtc wizard
   # or
   gtc wizard --answers ./my-create-answers.json
   ```

2. **provider を設定する**。これも対話型でも非対話型でも可能です:

   ```bash
   gtc setup ./my-bundle
   # or
   gtc setup ./my-bundle --answers ./my-setup-answers.json
   ```

3. **runtime を起動する**:

   ```bash
   gtc start ./my-bundle
   ```

</Steps>

capability flags や answers-document schemas を含む完全なリファレンスは、[gtc wizard](/ja/cli/wizard/) と [gtc setup](/ja/cli/setup/) を参照してください。

## 例: Hello World Flow

メッセージに応答するシンプルな flow を作成します:

```yaml title="flows/hello.ygtc"
name: hello_world
version: "1.0"
description: A simple greeting flow

nodes:
  - id: greet
    type: reply
    config:
      message: "Hello! I'm your digital worker."

triggers:
  - type: message
    pattern: "hello"
    target: greet
```

## Bundle Structure

一般的な Greentic bundle は次のようになります:

```
my-bundle/
├── greentic.demo.yaml      # Main configuration
├── providers/
│   └── messaging/
│       └── messaging-telegram.gtpack
├── apps/
│   └── my-app/
│       └── flows/
│           └── on_message.ygtc
└── seeds.yaml              # Seed data (optional)
```

## Configuration File

メイン設定ファイル（`greentic.demo.yaml`）はセットアップを定義します:

```yaml title="greentic.demo.yaml"
name: my-digital-worker
version: "1.0"

providers:
  messaging-telegram:
    pack: "providers/messaging/messaging-telegram.gtpack"
    setup_flow: "setup_default"

apps:
  my-app:
    path: "apps/my-app"
    default_flow: "on_message"

tenants:
  demo:
    name: Demo Tenant
    teams:
      default:
        channels:
          telegram:
            provider: messaging-telegram
```
