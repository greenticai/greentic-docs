Esta guía te lleva de cero a tener un digital worker de Greentic en funcionamiento en pocos minutos. La vía más rápida es lanzar una de las demos publicadas del catálogo [`greentic-demo`](https://github.com/greenticai/greentic-demo): cada demo usa el mismo flujo de tres pasos.

<Aside type="tip">
¿Quieres ver todas las demos disponibles y saber exactamente qué hace cada una? Ve a [Ejecutar Demos](/es/getting-started/running-demos/).
</Aside>

## Prerrequisitos

Antes de empezar, asegúrate de tener:

- **`gtc` CLI** instalado; consulta [Instalación](/es/getting-started/installation/)
- **Rust** 1.91 o posterior (solo si compilas desde el código fuente)
- Una **URL pública** para proveedores basados en webhook (por ejemplo, `ngrok` o `cloudflared`); no es necesaria para el quickstart solo con WebChat

## Lanzar la demo quickstart

La demo `quickstart` es el bundle end-to-end más pequeño: una tarjeta de bienvenida, una tarjeta de información y una interacción de eco sobre WebChat. No necesita cuentas externas.

<Steps>

1. **Crea el bundle** a partir del documento create-answers publicado:

   ```bash
   gtc wizard --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-create-answers.json
   ```

   El wizard descarga el pack de la demo, resuelve dependencias de capacidades (por ejemplo, añade `state-memory` automáticamente porque el proveedor WebChat necesita `greentic:state/state-store`) y escribe un directorio de bundle local.

2. **Configura los proveedores** con el documento setup-answers publicado:

   ```bash
   gtc setup ./quickstart-demo-bundle --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-setup-answers.json
   ```

   La configuración detecta automáticamente el tenant del bundle, guarda secretos por proveedor y valida que cada capacidad requerida esté satisfecha.

3. **Inicia el runtime**:

   ```bash
   gtc start ./quickstart-demo-bundle
   ```

   El runner imprime la URL pública de WebChat. Ábrela en un navegador y deberías ver la tarjeta de bienvenida.

</Steps>

<Aside type="note">
El mismo flujo de tres pasos funciona para todas las demos del catálogo. Para probar otra, solo sustituye el ID de la demo, por ejemplo `helpdesk-itsm`, `hr-onboarding` o `weather-mcp-demo`. Consulta [Ejecutar Demos](/es/getting-started/running-demos/) para ver la lista completa.
</Aside>

## Crear tu propio bundle

Cuando estés listo para ir más allá del catálogo publicado, se aplican los mismos tres comandos; simplemente consumen tus propios archivos en lugar de URLs de release:

<Steps>

1. **Crea un bundle** con el wizard. Puedes ejecutarlo de forma interactiva o con un documento local de respuestas:

   ```bash
   gtc wizard
   # or
   gtc wizard --answers ./my-create-answers.json
   ```

2. **Configura proveedores**, de nuevo de forma interactiva o no interactiva:

   ```bash
   gtc setup ./my-bundle
   # or
   gtc setup ./my-bundle --answers ./my-setup-answers.json
   ```

3. **Inicia el runtime**:

   ```bash
   gtc start ./my-bundle
   ```

</Steps>

Consulta [gtc wizard](/es/cli/wizard/) y [gtc setup](/es/cli/setup/) para ver la referencia completa, incluidas las banderas de capacidades y los esquemas de documentos de respuestas.

## Ejemplo: Flujo Hello World

Crea un flujo sencillo que responda a mensajes:

```yaml title="flows/hello.ygtc"
name: hello_world
version: "1.0"
description: A simple greeting flow

nodes:
  - id: greet
    type: reply
    config:
      message: "Hello! I'm your digital worker."

triggers:
  - type: message
    pattern: "hello"
    target: greet
```

## Estructura del Bundle

Un bundle típico de Greentic se ve así:

```
my-bundle/
├── greentic.demo.yaml      # Main configuration
├── providers/
│   └── messaging/
│       └── messaging-telegram.gtpack
├── apps/
│   └── my-app/
│       └── flows/
│           └── on_message.ygtc
└── seeds.yaml              # Seed data (optional)
```

## Archivo de Configuración

El archivo de configuración principal (`greentic.demo.yaml`) define tu entorno:

```yaml title="greentic.demo.yaml"
name: my-digital-worker
version: "1.0"

providers:
  messaging-telegram:
    pack: "providers/messaging/messaging-telegram.gtpack"
    setup_flow: "setup_default"

apps:
  my-app:
    path: "apps/my-app"
    default_flow: "on_message"

tenants:
  demo:
    name: Demo Tenant
    teams:
      default:
        channels:
          telegram:
            provider: messaging-telegram
```
