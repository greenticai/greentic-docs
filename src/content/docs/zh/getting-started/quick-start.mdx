本指南将帮助你在几分钟内从零开始运行一个 Greentic digital worker。最快的方式是启动 [`greentic-demo`](https://github.com/greenticai/greentic-demo) catalog 中已发布的一个 demo，每个 demo 都遵循相同的三步流程。

<Aside type="tip">
想浏览所有可用 demo，并准确了解每个 demo 的功能？请跳转到[运行 Demo](/zh/getting-started/running-demos/)。
</Aside>

## 先决条件

开始之前，请确保你已具备：

- 已安装 **`gtc` CLI**，参见[安装](/zh/getting-started/installation/)
- **Rust** 1.91 或更高版本（仅在从源码构建时需要）
- 用于 webhook providers 的**公开 URL**（例如 `ngrok` 或 `cloudflared`）；如果只是 WebChat quickstart，则不需要

## 启动 quickstart demo

`quickstart` demo 是最小的端到端 bundle：包含 welcome card、about card，以及通过 WebChat 进行的 echo 交互。它不需要任何外部账户。

<Steps>

1. **创建 bundle**，使用已发布的 create-answers 文档：

   ```bash
   gtc wizard --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-create-answers.json
   ```

   wizard 会下载 demo pack、解析 capability 依赖（例如由于 WebChat provider 需要 `greentic:state/state-store`，它会自动拉取 `state-memory`），并写出本地 bundle 目录。

2. **配置 providers**，使用已发布的 setup-answers 文档：

   ```bash
   gtc setup ./quickstart-demo-bundle --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-setup-answers.json
   ```

   setup 会自动检测 bundle 的 tenant，按 provider 持久化 secrets，并验证所有必需 capability 都已满足。

3. **启动 runtime**：

   ```bash
   gtc start ./quickstart-demo-bundle
   ```

   runner 会打印公开的 WebChat URL，在浏览器中打开它后，你应该能看到 welcome card。

</Steps>

<Aside type="note">
catalog 中的每个 demo 都适用同样的三步流程。若要尝试其他 demo，只需替换 demo ID，例如 `helpdesk-itsm`、`hr-onboarding` 或 `weather-mcp-demo`。完整列表请参见[运行 Demo](/zh/getting-started/running-demos/)。
</Aside>

## 构建你自己的 bundle

当你准备跳出已发布 catalog 时，仍然使用同样的三个命令，只是它们会读取你自己的文件，而不是 release URL：

<Steps>

1. **创建一个 bundle**，使用 wizard。你可以交互式运行，也可以使用本地 answers 文档：

   ```bash
   gtc wizard
   # or
   gtc wizard --answers ./my-create-answers.json
   ```

2. **配置 providers**，同样可以选择交互式或非交互式：

   ```bash
   gtc setup ./my-bundle
   # or
   gtc setup ./my-bundle --answers ./my-setup-answers.json
   ```

3. **启动 runtime**：

   ```bash
   gtc start ./my-bundle
   ```

</Steps>

完整参考请参见 [gtc wizard](/zh/cli/wizard/) 和 [gtc setup](/zh/cli/setup/)，其中包括 capability flags 和 answers 文档 schema。

## 示例：Hello World Flow

创建一个简单的 flow，用于响应消息：

```yaml title="flows/hello.ygtc"
name: hello_world
version: "1.0"
description: A simple greeting flow

nodes:
  - id: greet
    type: reply
    config:
      message: "Hello! I'm your digital worker."

triggers:
  - type: message
    pattern: "hello"
    target: greet
```

## Bundle 结构

一个典型的 Greentic bundle 如下所示：

```
my-bundle/
├── greentic.demo.yaml      # Main configuration
├── providers/
│   └── messaging/
│       └── messaging-telegram.gtpack
├── apps/
│   └── my-app/
│       └── flows/
│           └── on_message.ygtc
└── seeds.yaml              # Seed data (optional)
```

## 配置文件

主配置文件（`greentic.demo.yaml`）定义了你的 setup：

```yaml title="greentic.demo.yaml"
name: my-digital-worker
version: "1.0"

providers:
  messaging-telegram:
    pack: "providers/messaging/messaging-telegram.gtpack"
    setup_flow: "setup_default"

apps:
  my-app:
    path: "apps/my-app"
    default_flow: "on_message"

tenants:
  demo:
    name: Demo Tenant
    teams:
      default:
        channels:
          telegram:
            provider: messaging-telegram
```
