## 概述

**Greentic Setup** 是 Greentic bundle 的配置引擎。运行 `gtc setup <bundle>` 会在浏览器中打开一个交互式 Web 控制台，您可以在其中配置 provider 凭据、检查当前 scope 并为每个 provider 运行引导式设置向导——全部在本地托管的受保护单页应用程序中完成。

对于 CI 或 SSH 环境，`gtc setup --no-ui <bundle>` 会回退到终端对话框流程。

## 启动

```bash
# 启动 Web 控制台（默认）
gtc setup ./my-bundle

# 无头终端模式（CI / SSH）
gtc setup --no-ui ./my-bundle

# 无 UI 应用预构建的答案文件
gtc setup --answers answers.json ./my-bundle
```

运行 `gtc setup ./my-bundle` 时，二进制文件会：

1. 加载并验证您的 bundle 配置。
2. 在随机可用端口上启动绑定到 `127.0.0.1` 的本地 [Axum](https://docs.rs/axum) HTTP 服务器。
3. 生成仅存储在内存中的一次性 Bearer 令牌——永远不会写入磁盘。
4. 在默认浏览器中打开 `http://127.0.0.1:<port>/?token=<token>`。
5. 提供 Greentic Setup 单页应用程序。

关闭浏览器标签页或使用控制台中的 **Close** 按钮时，服务器会自动关闭。

## 控制台

Greentic Setup 控制台是使用 [Alpine.js](https://alpinejs.dev/) 构建的单页应用程序。它提供了简洁、极简的界面，无需离开浏览器即可管理 bundle 配置。

### 布局

控制台有三个主要区域：

- **顶栏** — 显示 bundle 名称和优雅关闭服务器的关闭按钮。
- **侧边栏** — 包含三个 scope 选择器（租户、环境、团队）和导航菜单。
- **主内容区** — 渲染当前活动视图。

### 侧边栏导航

侧边栏顶部的三个下拉选择器允许您更改活动 scope 上下文：

| 选择器 | 控制内容 |
|--------|---------|
| 租户 | 密钥和 provider 配置的租户命名空间 |
| 环境 | `demo`、`dev`、`staging` 或自定义值 |
| 团队 | 租户内的子团队 scope |

**Manage** 导航部分显示：

| 条目 | 状态 |
|------|------|
| Overview | Phase 1a 中可用 |
| Providers | Phase 1b 即将推出 |
| Secrets | Phase 1b 即将推出 |
| Capabilities | Phase 1b 即将推出 |

标记为 "1b" 的条目在当前版本中是只读占位符。

### Overview 视图

Overview 是默认的着陆视图。包含：

- 带有 bundle 显示名称的**欢迎标题**。
- **统计行**，显示四个摘要芯片：已配置的 Scope、找到的 Provider、已存储的密钥和警告。
- **Scope 列表** — 每个已配置 scope（租户/环境/团队组合）一个卡片，每个卡片显示在该 scope 中已设置的 provider 芯片。
- **空状态** — 如果尚未配置任何 scope，会显示空状态面板，提示您使用 **Add tenant** 按钮添加第一个租户。

## 向导

点击 **Add tenant**（或每个 scope 的 **Configure** 按钮）会启动内联设置向导。向导在控制台 shell 内运行——不会导航到单独的页面。

### 向导流程

向导通过一系列步骤收集配置：

<Steps>

1. **选择 scope** — 选择或确认租户、环境和团队。

2. **选择 provider** — 切换您要为此 scope 启用的 provider。

3. **配置每个 provider** — 为每个已启用的 provider 填写必填字段（API 密钥、公共 Webhook URL 等）。

4. **审核** — 在保存之前查看所有值的摘要。

5. **完成** — 凭据写入密钥存储；Overview 刷新以显示新 scope。

</Steps>

### 字段类型

向导支持以下输入类型：

| 类型 | 用途 |
|------|------|
| `text` | 普通字符串值（URL、ID、用户名） |
| `password` | 敏感值——渲染为掩码输入，完成后从内存中清除 |
| `textarea` | 多行值（证书 PEM 块、JSON 负载） |
| `select` | 枚举选项 |
| `switch` | 布尔开关（启用/禁用 provider） |

### 导航

每个步骤都有 **Back**、**Continue** 和 **Cancel** 操作。最后一步将 **Continue** 替换为 **Finish**。每步前进之前都会运行验证；错误会内联显示在相关字段下方。

## 无头模式

如果您通过 SSH 工作、在 CI 流水线中或禁用了浏览器启动，请传递 `--no-ui`：

```bash
gtc setup --no-ui ./my-bundle
```

这会调用旧版终端对话框流程。所有相同的配置值都会在提示符处交互式收集，密钥保存到与控制台向导相同的存储中。

### 答案文件

答案文件允许您在没有任何交互式提示的情况下自动化设置：

```bash
# 生成模板（仅必填字段）
gtc setup --dry-run --emit-answers answers.json ./my-bundle

# 生成包含可选字段的模板
gtc setup --dry-run --emit-answers answers.json --advanced ./my-bundle

# 应用答案
gtc setup --answers answers.json ./my-bundle
```

#### 演示目录答案

[`greentic-demo`](https://github.com/greenticai/greentic-demo) 目录中的每个演示都会发布一个 `*-setup-answers.json` 文档。直接应用它：

```bash
gtc setup ./quickstart-demo-bundle \
  --answers https://github.com/greenticai/greentic-demo/releases/latest/download/quickstart-setup-answers.json
```

有关包含描述和所需凭据的完整目录，请参阅[运行演示](/zh/getting-started/running-demos/)。

<Aside type="note">
Greentic Setup 从 `tenants/` 目录自动检测 bundle 的租户，因此即使 bundle 不是使用默认的 `demo` 租户创建的，密钥也会写入正确的租户 URI。它还对 bundle 中的每个 pack 运行**依赖项能力验证**，如果任何必需的能力（例如 `greentic:state/state-store`）仍未满足，则会发出警告。
</Aside>

## 安全性

Greentic Setup 对所有本地 UI 会话采用安全优先的方法：

| 控制 | 实现 |
|------|------|
| 仅回环 | 服务器仅绑定到 `127.0.0.1`——无网络暴露 |
| 随机端口 | 启动时选择随机可用端口——无固定端口可猜测 |
| Bearer 令牌 | 在内存中生成的一次性令牌，在首次加载时通过 URL 查询参数传递，然后仅存储在浏览器会话中 |
| Origin 强制 | 每个 API 请求都针对 `Origin` 允许列表进行验证；跨域请求被拒绝 |
| 日志中无密钥 | 凭据值从不写入 stdout、stderr 或日志文件 |
| 磁盘上无密钥 | Bearer 令牌和向导答案仅保存在内存中；使用后对敏感结构调用 `Zeroize` |
| 同源资源 | 所有 JS、CSS 和字体资源都从同一本地服务器提供——无外部 CDN 调用 |
| 无遥测 | 控制台不进行任何出站网络调用 |

## Bundle 能力

`bundle.yaml` 文件可以声明在设置期间解锁额外行为的能力。

### Bundle 资源（`greentic.cap.bundle_assets.read.v1`）

声明此能力后，Greentic Setup 会在 bundle 根目录创建 `assets/` 目录，并将 provider pack 资源复制到其中（例如 `assets/webchat-gui/`）。然后您可以自由修改这些文件；bundle 级别的资源文件在运行时会覆盖 pack 默认值。

```yaml title="bundle.yaml"
capabilities:
  - greentic.cap.bundle_assets.read.v1
```

<Aside type="tip">
设置创建资源后，当运行时加载它们时，您的 bundle 级别自定义项优先于 pack 默认值。
</Aside>

## 路线图

Greentic Setup Phase 1a 提供控制台 shell、Overview 视图和内联向导。**Phase 1b** 中即将推出的工作包括：

- **Providers** — 浏览 bundle 中的所有 pack，启用或禁用单个 provider，重新运行单个 provider 的设置而不影响其他 provider。
- **Secrets** — 密钥存储的完整 CRUD：按 scope 创建、读取（掩码）、更新和删除单个密钥条目。
- **Capabilities** — 按 pack 切换能力标志并检查依赖关系图。
- **重建触发器** — 从控制台请求正在运行的 bundle 热重载，无需重启完整运行时。

## 故障排除

### 浏览器未打开

如果您的系统不支持自动浏览器启动，请复制打印到终端的 URL 并手动粘贴到浏览器中：

```
Greentic Setup running at http://127.0.0.1:52341/?token=...
```

### Webhook 注册失败

```
Error: Failed to register webhook: 401 Unauthorized
```

检查您的 bot 令牌是否正确且未过期。

### 缺少必填字段

```
Error: Missing required field 'bot_token' for messaging-telegram
```

确保所有必填字段在向导中填写或存在于您的答案文件中。

### 公共 URL 无法访问

```
Error: Webhook verification failed: Connection refused
```

您的公共 URL 必须可从互联网访问。检查 ngrok 或 cloudflared 是否正在运行。

### 找不到凭据

```
Error: Secret 'slack_bot_token' not found
```

运行 `gtc setup` 配置 provider，或检查您的答案文件。
